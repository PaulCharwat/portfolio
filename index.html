Bootstrap

Was ist Bootstrap?

Bootstrap ist ein freies Frontend-CSS-Framework. Es enthält vorgefertigte Design-Vorlagen für Typografie, Formulare, Buttons, Tabellen, Grid-Systeme, Navigations- und andere Oberflächengestaltungselemente, sowie zusätzliche, optionale JavaScript-Erweiterungen.

Wie wird Bootstrap verwendet?

Bootstrap kann auf 2 verschiedene Arten verwendet werden.

1. Bootstrap herunterladen und in das Projekt einbinden

Bootstrap kann auf der offiziellen Website heruntergeladen werden.
https://getbootstrap.com/docs/5.1/getting-started/download/

Nach dem Download kann Bootstrap in das Projekt wie jede andere CSS oder JavaScript Datei eingebunden werden.

2. Bootstrap über CDN einbinden

Bootstrap kann auch über ein CDN (Content Delivery Network) eingebunden werden. Dadurch entfällt das herunterladen und einbinden der Bootstrap Dateien in das Projekt. Dazu muss folgender Code in den head der HTML-Datei eingefügt werden:







Bootstrap Grid System

Das Bootstrap Grid System ist ein System, dass es ermöglicht, die Website responsive zu gestalten. Es arrangiert die Spalten des Grid-Systems genau so, dass die optimale Anzahl an Spalten für die Displaybreite angezeigt werden.

Das Grid System besteht aus 12 Spalten. Diese können beliebig auf die Website verteilt werden.







Funktionsweise

Da Bootstrap ein CSS-Framework ist, wird es über CSS gesteuert. Das heißt alle Anpassungen des Designs werden über CSS Klassen vorgenommen.
Sobald also eine Bootstrap Klasse in ein HTML-Element eingefügt wird, wird das Design des Elements entsprechend angepasst.


Komponenten

Bootstrap bietet außerdem viele vorgefertigte Komponenten, die einfach in die Website eingebunden werden können. Der Code für diese Komponenten besteht meist dur aus "div"-Elementen die durch das integrierte Bootstrap CSS stark angepasst werden.

Erweiterte Funktionen der Komponenten werden durch JavaScript ermöglicht. Die benötigte Datei wird ebenso wie die CSS Datei bereitgestellt. Die Installation ist oben erklärt.

Die sechste Vorlesung behandelt Das Document Objekt Model (DOM) und den einstieg in jQuerry

Vorlesung 6

In der sechsten Vorlesung haben wir uns mit dem Document Objekt Model (DOM) beschäftigt. Das DOM ist eine Programmierschnittstelle, die es ermöglicht, auf HTML-Elemente zuzugreifen und diese zu verändern.
Das verändern eines HTML Elements passiert in zwei wesentlichen Schritten:
  • Zuerst wird das Element ausgewählt. Dies geschieht mit Hilfe von Selektoren.
  • Anschließend können Aktionen auf das Element angewand werden um es zu verändern.

Das Auswählen eines Elements geschieht mit Hilfe der Methode getElementById(). Diese Methode gibt ein Objekt zurück, dass das ausgewählte Element repräsentiert.

Die Änderungsmöglichkeiten sind ab diesem Punkt nahezu endlos. Es können Klassenattribute geändert werden, Elemente können gelöscht oder hinzugefügt werden und vieles mehr.


Das DOM

Das DOM ist ein Modell, das den Webbrowsern ermöglicht, HTML-Tags, Attribute und Inhalte einer Seite zu repräsentieren. Durch das DOM kann JavaScript mit dem HTML einer Seite kommunizieren und es z.B. ändern.

Es gibt verschiedene Methoden, um auf Elemente zuzugreifen, darunter getElementById() und getElementsByTagName(). Die innerHTML-Eigenschaft ermöglicht den Zugriff auf den HTML-Inhalt eines Elements.

Das DOM hat jedoch einige Schwächen, wie unterschiedliche Interpretationen in verschiedenen Browsern und die zeitaufwendige Navigation zwischen Knoten. Es gibt auch browserübergreifende Probleme bei der Standardisierung von DOM-Funktionen.

Die innerHTML-Eigenschaft erleichtert das Hinzufügen oder Ändern von Inhalten auf einer Seite, ist aber nicht standardmäßig im DOM enthalten. Sie repräsentiert den gesamten HTML-Inhalt eines Elements.

Das Überwinden von browserübergreifenden JavaScript-Problemen kann komplex sein, aber fortgeschrittene JavaScript-Programme können helfen, browserübergreifende Konsistenz zu gewährleisten.



Bibilotheken

Bibliotheken sind Sammlungen von Code, die von anderen Entwicklern erstellt und in eigenen Projekten verwendet werden können. Sie bieten effiziente Möglichkeiten, Details zu umgehen oder zu erleichtern.

In dieser Vorlesung haben wir uns mit der Bibliothek jQuerry beschäftigt. jQuerry ist eine JavaScript Bibliothek, die das DOM vereinfacht.
Um jQuerry zu verwenden muss es in die HTML Datei eingebunden werden. Das erfolgt entweder durch das herunterladen der Bibliothek und einbinden in die HTML Datei oder durch das einbinden der Bibliothek über einen CDN.

jQuery bietet leistungsstarke Selektionsmöglichkeiten, die CSS-Selektoren ähneln. Durch die Verkettung von Methoden können Aktionen auf ausgewählte Elemente angewendet werden. Die Bibliothek ermöglicht auch das Arbeiten mit Klassen. Hinzufügen, Entfernen und Umschalten von Klassen ist möglich.

Die attr()-Funktion von jQuery wird verwendet, um HTML-Attribute zu lesen oder festzulegen. Die removeAttr()-Funktion entfernt ein Attribut von einem Element. jQuery bietet auch Funktionen wie addClass(), removeClass() und toggleClass() für die Arbeit mit Klassen.

Die each()-Funktion von jQuery ermöglicht die Durchführung von Aktionen für jedes Element in einer Auswahl. Anonyme Funktionen werden als Argumente an each() übergeben.

Vorlesung 7

Die siebte Vorlesung thematisiert Event-Handling im jQuerry-Event-Modell. Das Event-Modell ist ein Modell, das beschreibt, wie Ereignisse in einer Anwendung verarbeitet werden.

Beim Aufrufen und Nutzen einer Website werden im Hintergrund verschiedene Ereignisse ausgelöst. Diese Ereignisse können mit JavaScript abgefangen und verarbeitet werden. Dieses Prinzip nennt mann ereignisgesteuerte Programmierung.

Ausgelöste Ereignisse werden in der Event-Queue gespeichert. Diese Ereignisse werden dann von dem Event-Loop an Event-Handler weitergeleitet.



Um bei Benutzerinteraktionen innerhalb einer Website eine Funktion auslösen zu können, sind drei Schritte notwendig:
  1. Elementauswahl: Das gewünschte HTML-Element muss ausgewählt werden, z.B. "th"-Element für eine Tabellenspaltenüberschrift.
  2. Eventbindung: Festlegen, welches Ereignis abgefangen werden soll, z.B. Mausklicks, Tastatureingaben, usw.
  3. Funktionsdefinition: Die Funktion, die aufgerufen werden soll, wenn das Ereignis eintritt, wird festgelegt. Zum Beispiel kann bei einem Klick auf eine Schaltfläche eine bestimmte Funktion aufgerufen werden.

Events repräsentieren Momente, in denen etwas auf der Webseite passiert. Beim Drücken der Maustaste wird beispielsweise das mousedown-Event ausgelöst, gefolgt von mouseup und schließlich click.

Maus-Events spielen eine wichtige Rolle, wie click für Mausklicks, dblclick für Doppelklicks, mousedown für den Moment des Maustastendrucks, mouseup für den Moment des Loslassens und scroll für Scroll-Aktionen. Das unload-Event wird ausgelöst, wenn der Benutzer die Seite verlässt, und ermöglicht letzte Aktionen vor dem Schließen.

Weitere Mouse-Events sind in untensetehnden Bild zu erkennen:



Formular-Events, spielen zudem eine zentrale Rolle in der Webentwicklung. HTML-Formulare ermöglichen Benutzern die Eingabe von Informationen, und durch die Verwendung von Formular-Events wie submit, reset, change, focus und blur können Interaktionen gezielt gesteuert werden. Zum Beispiel wird das submit-Event ausgelöst, wenn ein Benutzer ein Formular abschickt, und kann für die Validierung von Eingaben genutzt werden.

Ein weiterer wichtiger Aspekt sind Tastatur-Events, die die Verfolgung von Tastatureingaben ermöglichen. Mit keypress, keydown und keyup kann auf Tastatureingaben reagiert werden. Verschiedene Browser Tastatur-Events müssen unterschiedlich behandelt werden, was komplex werden kann.

Events können auch mit Funktionen verknüpft werden. Dies kann entweder inline also im HTML Code geschehen, was die einfachste lösung ist, oder im Script Bereich was für die Übersichtlichkeit förderlich ist.

Wie alles in der IT entwickelt sich auch das Event-Modell ständig weiter. Es ein "traditionelles" Modell und ein "modernes" Modell.

Das traditionelle Modell wird als eine Methode beschrieben, um Event-Handler einem Tag zuzuweisen. Hierbei wurde ein Beispiel mit window.onload gegeben, um eine Funktion nach dem Laden der Seite auszuführen. Betont wurde jedoch, dass diese Methode Einschränkungen hat, insbesondere wenn es darum geht, mehrere Funktionen für dasselbe Event desselben Tags zuzuweisen.



Die moderne Methode, bekannt als Event-Listener, ist eine verbesserte Alternative. Diese Methode ermöglicht es, mehrere Funktionen für dasselbe Event desselben Tags zuzuweisen. Das im Vergleich zum traditionellen Modell deutlich flexibler.

Es gibt allerdings noch eine Methode, und zwar über jQuerry. jQuerry erleichtert die erstellung von Event-Listenern deutlich, da die unterschiede zwischen den verschiedenen Browsern ignoriert werden können. AJAX revolutionierte die Art und Weise, wie Webanwendungen mit ihrem Server kommunizierten. Es mussten nicht mehr gesamte Websiten neu geladen werden wenn sich eine Zahl geändert hat. Jetzt konnte individuell das geladen werden was benötigt wurde.


Assynkorne Programmierung bringt auch so viele Vorteile. So können z.B. Systeme weiterlaufen wärend nur ein Bestandteil gewartet wird. Außerdem veringern sich die Lade- und Wartezeiten für den Nutzer drastisch. Die folgende Grafik zeigt die Zeitersparnis im Vergleich zu einer synchronen Programmierung:



Durch die Einführung von Promises wird die Arbeit mit asyncronen Aktionen deutlich vereinfacht. Promises sind Objekte, die eine asynchrone Aktion darstellen. Sie können entweder erfüllt oder abgelehnt werden.

Außerdem wird die Verwendung von "await" in asyncronen Funktionen eingeführt. "await" ermöglicht es Prommises asyncronen Code wie syncronen Code auszuführen. Dies reduziert die Komplexität.


Die Implementierung von AJAX erfolgt durch JavaScript, welches Anfrage-Objekte erstellt und an den Server sendet. Hierbei wird am Beispiel eines PHP-Skripts, das nur die benötigten Daten anstelle einer ganzen HTML-Seite zurückgibt, veranschaulicht, wie effiziente Datenübertragung möglich ist.

Die Schritte umfassen:
  1. Erzeugung der Anfrage: Der Client sendet Anfragen an den Server und wartet nicht auf die vollständige Antwort, was die Benutzerinteraktion verbessert, da der Nutzer die Seite weiterhin verwenden kann.
  2. Serververarbeitung: Der Server führt externe Komponenten aus und sendet Daten inkrementell zurück. Dies ermöglicht das effizientes Aktualisieren der Webseite.
  3. Integration der Daten: Der Client kann asynchron Daten vom Server in die Webseite einbauen, was besonders für das Nachladen größerer Daten wie Bilder oder das ständige Aktualisieren bestimmter Datenfelder geeignet ist.

In der sechsten Vorlesung haben wir uns mit dem Document Objekt Model (DOM) beschäftigt. Das DOM ist eine Programmierschnittstelle, die es ermöglicht, auf HTML-Elemente zuzugreifen und diese zu verändern.
Das verändern eines HTML Elements passiert in zwei wesentlichen Schritten:
  • Zuerst wird das Element ausgewählt. Dies geschieht mit Hilfe von Selektoren.
  • Anschließend können Aktionen auf das Element angewand werden um es zu verändern.

Das Auswählen eines Elements geschieht mit Hilfe der Methode getElementById(). Diese Methode gibt ein Objekt zurück, dass das ausgewählte Element repräsentiert.

Die Änderungsmöglichkeiten sind ab diesem Punkt nahezu endlos. Es können Klassenattribute geändert werden, Elemente können gelöscht oder hinzugefügt werden und vieles mehr.


Das DOM

Das DOM ist ein Modell, das den Webbrowsern ermöglicht, HTML-Tags, Attribute und Inhalte einer Seite zu repräsentieren. Durch das DOM kann JavaScript mit dem HTML einer Seite kommunizieren und es z.B. ändern.

Es gibt verschiedene Methoden, um auf Elemente zuzugreifen, darunter getElementById() und getElementsByTagName(). Die innerHTML-Eigenschaft ermöglicht den Zugriff auf den HTML-Inhalt eines Elements.

Das DOM hat jedoch einige Schwächen, wie unterschiedliche Interpretationen in verschiedenen Browsern und die zeitaufwendige Navigation zwischen Knoten. Es gibt auch browserübergreifende Probleme bei der Standardisierung von DOM-Funktionen.

Die innerHTML-Eigenschaft erleichtert das Hinzufügen oder Ändern von Inhalten auf einer Seite, ist aber nicht standardmäßig im DOM enthalten. Sie repräsentiert den gesamten HTML-Inhalt eines Elements.

Das Überwinden von browserübergreifenden JavaScript-Problemen kann komplex sein, aber fortgeschrittene JavaScript-Programme können helfen, browserübergreifende Konsistenz zu gewährleisten.



Bibilotheken

Bibliotheken sind Sammlungen von Code, die von anderen Entwicklern erstellt und in eigenen Projekten verwendet werden können. Sie bieten effiziente Möglichkeiten, Details zu umgehen oder zu erleichtern.

In dieser Vorlesung haben wir uns mit der Bibliothek jQuerry beschäftigt. jQuerry ist eine JavaScript Bibliothek, die das DOM vereinfacht.
Um jQuerry zu verwenden muss es in die HTML Datei eingebunden werden. Das erfolgt entweder durch das herunterladen der Bibliothek und einbinden in die HTML Datei oder durch das einbinden der Bibliothek über einen CDN.

jQuery bietet leistungsstarke Selektionsmöglichkeiten, die CSS-Selektoren ähneln. Durch die Verkettung von Methoden können Aktionen auf ausgewählte Elemente angewendet werden. Die Bibliothek ermöglicht auch das Arbeiten mit Klassen. Hinzufügen, Entfernen und Umschalten von Klassen ist möglich.

Die attr()-Funktion von jQuery wird verwendet, um HTML-Attribute zu lesen oder festzulegen. Die removeAttr()-Funktion entfernt ein Attribut von einem Element. jQuery bietet auch Funktionen wie addClass(), removeClass() und toggleClass() für die Arbeit mit Klassen.

Die each()-Funktion von jQuery ermöglicht die Durchführung von Aktionen für jedes Element in einer Auswahl. Anonyme Funktionen werden als Argumente an each() übergeben.

Vorlesung 7

Die siebte Vorlesung thematisiert Event-Handling im jQuerry-Event-Modell. Das Event-Modell ist ein Modell, das beschreibt, wie Ereignisse in einer Anwendung verarbeitet werden.

Beim Aufrufen und Nutzen einer Website werden im Hintergrund verschiedene Ereignisse ausgelöst. Diese Ereignisse können mit JavaScript abgefangen und verarbeitet werden. Dieses Prinzip nennt mann ereignisgesteuerte Programmierung.

Ausgelöste Ereignisse werden in der Event-Queue gespeichert. Diese Ereignisse werden dann von dem Event-Loop an Event-Handler weitergeleitet.



Um bei Benutzerinteraktionen innerhalb einer Website eine Funktion auslösen zu können, sind drei Schritte notwendig:
  1. Elementauswahl: Das gewünschte HTML-Element muss ausgewählt werden, z.B. "th"-Element für eine Tabellenspaltenüberschrift.
  2. Eventbindung: Festlegen, welches Ereignis abgefangen werden soll, z.B. Mausklicks, Tastatureingaben, usw.
  3. Funktionsdefinition: Die Funktion, die aufgerufen werden soll, wenn das Ereignis eintritt, wird festgelegt. Zum Beispiel kann bei einem Klick auf eine Schaltfläche eine bestimmte Funktion aufgerufen werden.

Events repräsentieren Momente, in denen etwas auf der Webseite passiert. Beim Drücken der Maustaste wird beispielsweise das mousedown-Event ausgelöst, gefolgt von mouseup und schließlich click.

Maus-Events spielen eine wichtige Rolle, wie click für Mausklicks, dblclick für Doppelklicks, mousedown für den Moment des Maustastendrucks, mouseup für den Moment des Loslassens und scroll für Scroll-Aktionen. Das unload-Event wird ausgelöst, wenn der Benutzer die Seite verlässt, und ermöglicht letzte Aktionen vor dem Schließen.

Weitere Mouse-Events sind in untensetehnden Bild zu erkennen:



Formular-Events, spielen zudem eine zentrale Rolle in der Webentwicklung. HTML-Formulare ermöglichen Benutzern die Eingabe von Informationen, und durch die Verwendung von Formular-Events wie submit, reset, change, focus und blur können Interaktionen gezielt gesteuert werden. Zum Beispiel wird das submit-Event ausgelöst, wenn ein Benutzer ein Formular abschickt, und kann für die Validierung von Eingaben genutzt werden.

Ein weiterer wichtiger Aspekt sind Tastatur-Events, die die Verfolgung von Tastatureingaben ermöglichen. Mit keypress, keydown und keyup kann auf Tastatureingaben reagiert werden. Verschiedene Browser Tastatur-Events müssen unterschiedlich behandelt werden, was komplex werden kann.

Events können auch mit Funktionen verknüpft werden. Dies kann entweder inline also im HTML Code geschehen, was die einfachste lösung ist, oder im Script Bereich was für die Übersichtlichkeit förderlich ist.

Wie alles in der IT entwickelt sich auch das Event-Modell ständig weiter. Es ein "traditionelles" Modell und ein "modernes" Modell.

Die letzte Vorlesung handelt von dem thema Asynchrone Programmierung - AJAX

11.12.2023

Vorlesung 8

Die letzte Vorlesung handelt von dem Thema Asynchrone Programmierung - AJAX. AJAX steht für Asynchronous JavaScript and XML.

AJAX ermöglicht es Webseiten, Inhalte asynchron durch asynchrone Datenübertragung zwischen Browser und Webserver zu aktualisieren. Dies bedeutet, dass es möglich ist, Daten vom Server abzurufen und anzuzeigen, ohne die Seite neu zu laden.
Das Bild zeigt die Kommunikation zwischen Server und Client in eine dynamischen Aufbau:



AJAX revolutionierte die Art und Weise, wie Webanwendungen mit ihrem Server kommunizierten. Es mussten nicht mehr gesamte Websiten neu geladen werden wenn sich eine Zahl geändert hat. Jetzt konnte individuell das geladen werden was benötigt wurde.


Assynkorne Programmierung bringt auch so viele Vorteile. So können z.B. Systeme weiterlaufen wärend nur ein Bestandteil gewartet wird. Außerdem veringern sich die Lade- und Wartezeiten für den Nutzer drastisch. Die folgende Grafik zeigt die Zeitersparnis im Vergleich zu einer synchronen Programmierung:



Durch die Einführung von Promises wird die Arbeit mit asyncronen Aktionen deutlich vereinfacht. Promises sind Objekte, die eine asynchrone Aktion darstellen. Sie können entweder erfüllt oder abgelehnt werden.

Außerdem wird die Verwendung von "await" in asyncronen Funktionen eingeführt. "await" ermöglicht es Prommises asyncronen Code wie syncronen Code auszuführen. Dies reduziert die Komplexität.


Die Implementierung von AJAX erfolgt durch JavaScript, welches Anfrage-Objekte erstellt und an den Server sendet. Hierbei wird am Beispiel eines PHP-Skripts, das nur die benötigten Daten anstelle einer ganzen HTML-Seite zurückgibt, veranschaulicht, wie effiziente Datenübertragung möglich ist.

Die Schritte umfassen:
  1. Erzeugung der Anfrage: Der Client sendet Anfragen an den Server und wartet nicht auf die vollständige Antwort, was die Benutzerinteraktion verbessert, da der Nutzer die Seite weiterhin verwenden kann.
  2. Serververarbeitung: Der Server führt externe Komponenten aus und sendet Daten inkrementell zurück. Dies ermöglicht das effizientes Aktualisieren der Webseite.
  3. Integration der Daten: Der Client kann asynchron Daten vom Server in die Webseite einbauen, was besonders für das Nachladen größerer Daten wie Bilder oder das ständige Aktualisieren bestimmter Datenfelder geeignet ist.

Die Umsetzung erfolgt durch JavaScript-Funktionen wie getVerkaufteBoards(), die Anfrage-Objekte erstellen und an das PHP-Skript senden. Der Server liefert nur die relevanten Daten zurück, und die Webseite wird durch JavaScript und Dynamic HTML aktualisiert.
